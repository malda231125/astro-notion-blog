## **2. Why Specialized Agents are Superior (How I Built an OpenClaw Superteam)**

- **채널명**: Riley Brown
- **영상 제목**: Why Specialized Agents are Superior (How I Built an OpenClaw Superteam)

- **영상 길이**: 18:16

### **영상 내용**


Riley Brown은 2주간 수백 개의 AI 에이전트 워크플로를 구축한 경험을 바탕으로, **좁은 범위에 특화된(narrow) 에이전트 팀**이 미래라고 주장한다.


OpenClaw, Manis, Clawed Code, Perplexity Computer 4가지 도구를 비교 테스트한 결과, Manis와 Perplexity Computer는 각 작업마다 별도의 클라우드 컴퓨터를 제공하는 "커맨드 센터" 방식인 반면, **OpenClaw는 하나의 컴퓨터에서 돌아가는 AI 에이전트에 뛰어난 메모리, 스킬 시스템, 그리고 Telegram·WhatsApp·Discord·Slack 등 다양한 메신저 연동 게이트웨이**를 제공해 차별화된다고 설명한다.


핵심 발견은 **스킬이 많아질수록 에이전트 신뢰성이 떨어진다**는 점이다. 스킬 30개 이상의 범용 에이전트보다 **7~10개 스킬의 특화 에이전트**가 최적의 성능을 보인다. 이에 따라 15개의 좁은 범위 에이전트 팀을 구축하는 전략을 채택했다.


구체적 사례로 YouTube 전용 에이전트를 소개한다. 이 에이전트는 구독자·조회수·전환율 3가지 목표에만 집중하며, YouTube 리서치(SER API + Supera Data), 썸네일 생성(Nano Banana + 본인 사진), Notion 스크립트 관리 등 3가지 핵심 스킬만 탑재했다. 매일 아침 경쟁자 썸네일을 스크랩하여 아이디어를 제안하는 자동화도 수행한다.


특화 에이전트의 장점으로 **쉬운 복제**(YouTube→TikTok 에이전트 전환), **팀 공유 용이성**, **명확한 의도(intent) 부여** 가능을 꼽는다. OpenAI 전 임시 CEO Emmett Shear의 "프롬프트 시대는 끝났고 이제 모델에 의도(intent)를 부여하는 시대"라는 트윗을 인용하며, 범용 에이전트는 명확한 의도를 부여하기 어렵다고 강조한다.

## **3. Stop Sleeping on These 5 Free OpenClaw Tools**

- **영상 길이**: 11:23

### **영상 내용**


OpenClaw 에이전트와 함께 사용하면 강력한 **무료 도구 5가지**를 소개하는 실용적인 영상이다.


**1) AgentMail** (agentmail.to) — AI 에이전트 전용 이메일 서비스. Gmail의 봇 차단 문제를 우회하여 에이전트가 독립적인 이메일을 보유하고 로그인·이메일 발송을 수행할 수 있다. API 키만 에이전트에 전달하면 자동으로 설정된다.


**2) Here.now** — 에이전트를 통한 즉시 웹 호스팅 서비스. "Hey, Max, host this to Here.now"라고 말하면 고유 URL이 생성되어 마케팅 플랜, 문서 등을 즉시 공유할 수 있다. 무료 계정 시 영구 호스팅 가능.


**3) Tavily** (tavly.com) — 에이전트 전용 웹 검색 도구. Brave 브라우저가 유료화된 대안으로, 에이전트에 최적화된 빠른 검색을 월 1,000회 무료로 제공한다.


**4) Firecrawl** (firecrawl.dev) — 웹사이트를 AI가 처리하기 좋은 깨끗한 데이터로 변환하는 스크래핑 도구. 경쟁사 제품·가격 분석, 웹사이트 복제 등에 활용. Tavily로 페이지를 찾고 Firecrawl로 내용을 추출하는 조합이 효과적이다.


**5) Remotion** — 에이전트가 프로그래밍 방식으로 영상을 제작하는 도구. 음성으로 "프로모 영상 만들어줘"라고 지시하면 편집 없이 전문적인 영상을 자동 생성한다. 무료 버전으로 충분하며, 대화형으로 색상·크기·텍스트를 수정할 수 있다.

**인프라 구성**: 개인 컴퓨터 대신 보안상 안전한 Hostinger VPS($6.29/월)에 OpenClaw을 원클릭 배포한다. 로컬 Mac Mini 실행은 보안 위험이 있으므로 VPS를 권장한다.


**모델 라우팅 전략**: Open Router를 단일 API 키로 설정하여 용도별로 다른 모델을 사용한다. 일반 질문/브레인스토밍에는 **Minimax 2.5**(매우 저렴), 코딩에는 **DeepSeek V3.2** 또는 **GLM5**, 복잡한 작업에는 **Claude Opus 4.6**, 글쓰기에는 **Claude Sonnet 4.5**를 배정한다. 기본 GPT 5.2 대비 **70~80% 비용 절감**이 가능하다.


**추가 최적화 4가지**:

1. **스마트 하트비트**: 기본 30분 → 55~60분 간격으로 변경, 가장 저렴한 flashlight 모델 사용
2. **컨텍스트 윈도우 제한**: 15회 교환 또는 30분마다 세션 리셋하여 토큰 누적 방지 → 오버헤드 80% 감소
3. **환경 변수 보안**: Docker 환경변수로 API 키 관리하여 설정 파일 노출 방지
4. **모델 즉시 전환**: `/model sonnet` 명령으로 작업 중 실시간 모델 변경

결과적으로 하루 10센트 미만, 월 $2 수준으로 콘텐츠 에이전트·제품 에이전트·리서치 에이전트·개발 에이전트 등 19개 에이전트 플릿을 운영하고 있으며, 자동으로 웹 앱(포모도로 타이머, 프롬프트 매니저, 썸네일 분석기 등)을 구축하게 한다.

OpenClaw **2026.2.06 업데이트** 주요 변경사항을 상세히 분석한다.


**1) OpenClaw Secrets (외부 시크릿 관리)**: API 키·비밀번호·토큰을 하나의 보안 저장소에서 관리하는 기능 추가. 기존에는 설정 파일에 하드코딩하던 위험한 방식을 개선하여, 다수 에이전트가 안전하게 인증 정보를 참조할 수 있게 되었다.


**2) ACP Threadbound Agents**: 에이전트 커뮤니케이션 프로토콜(ACP)의 스레드 바인딩이 1급 런타임 기능으로 승격. 각 에이전트가 특정 대화 스레드에 고정되어 컨텍스트 혼동 없이 멀티 에이전트 워크플로를 수행할 수 있다.

**4) Agent Routing CLI (bind/unbind)**: 명령줄에서 에이전트를 특정 라우트/작업에 바인딩·언바인딩하는 기능. 새 워크플로 테스트 시 에이전트를 유연하게 교체할 수 있으며, 스크립팅과 버전 관리가 가능하다.


**5) Android 앱 개선** 및 **11건의 보안 강화 패치**: 모바일 관리 기능 향상, 대규모 보안 감사 후 수정 반영으로 비즈니스 수준의 신뢰성 확보.

Microsoft가 출시한 **Copilot Tasks**와 OpenClaw를 비교 분석한다. Copilot Tasks는 단순 질의응답이 아니라 실제 작업을 수행하는 에이전트로, "내 한 주를 계획해줘"라고 지시하면 캘린더에 회의를 잡고, 이메일을 작성하고, 의제를 만드는 등의 작업을 자동 처리한다.

**Copilot Tasks의 강점**: Outlook·Word·Excel·PowerPoint 등 Microsoft 생태계 완벽 통합, 동의 기반 시스템으로 안전성 확보, 클라우드 실행으로 로컬 기기 부담 없음. **약점**: 현재 제한적 리서치 프리뷰 단계(대기 명단), Microsoft 외부 도구 연동 범위 불명확, 완전 자동화는 아직 미완성.


결론적으로 Copilot Tasks는 Microsoft 생태계 사용자에게 강력한 대안이지만, OpenClaw의 오픈소스 자유도와 플랫폼 독립성에는 미치지 못한다고 평가한다.

## **11. Toggle + OpenClaw Is INSANE!**

### **영상 내용**


Chrome 확장 프로그램 **Toggle X**와 OpenClaw의 결합이 "맹목적 에이전트 문제(blind agent problem)"를 해결한다고 소개한다.


**문제**: 기존 AI 에이전트는 매번 컨텍스트를 잃어 "어떤 프로젝트인가요?", "지난번 뭘 논의했나요?"를 반복적으로 물어봐야 했다.


**해결**: Toggle X는 Chrome에서 백그라운드로 실행되며 ① 탭 활동(어떤 사이트를 방문하는지) ② 집중 시간(얼마나 오래 보는지) ③ 의도 추론(브라우징 패턴 분석)을 구조화된 데이터로 수집한다. OpenClaw 에이전트가 이 데이터를 전용 스킬을 통해 참조하면, 사용자가 무엇을 하고 있는지 이미 파악한 상태에서 작업을 수행한다.


**자동화 핵심**: 단순히 실시간 컨텍스트만 제공하는 것이 아니라, 주 단위 패턴을 학습한다. 매주 반복하는 콘텐츠 성과 확인 → 라운드업 작성 → 멤버 질문 팔로업 → 콘텐츠 캘린더 업데이트 같은 워크플로를 감지하면, OpenClaw가 자동으로 처리하기 시작한다.


**바이럴 데모 2가지**: ① Notion 에이전트 — Toggle이 최근 작업한 Notion 페이지를 파악하여 "이번 주 작업 기반으로 콘텐츠 플랜 업데이트해줘"만 말하면 자동 처리. ② 영상 워크플로 — 영상 완성 후 항상 Google Sheet 기록 → Notion 업데이트 → Discord 팀 메시지 순서를 반복하면, OpenClaw가 이 3단계를 자동 실행한다.


셀프 호스팅 + 로컬 데이터 처리라는 프라이버시 장점도 강조한다.

## **12. ZeroClaw vs OpenClaw: It's Not Even Close**

- **채널명**: jordanUrbsAI
- **영상 제목**: ZeroClaw vs OpenClaw: It's Not Even Close
- **조회수**: 1,237
- **영상 URL**: [https://www.youtube.com/watch?v=Xv_Mown2hDA](https://www.youtube.com/watch?v=Xv_Mown2hDA)
- **영상 길이**: 12:40

### **영상 내용**


신생 AI 에이전트 프레임워크 **ZeroClaw**를 직접 설치·사용한 후 OpenClaw와 비교하는 솔직한 리뷰이다.


**기술 비교**:

- OpenClaw: TypeScript/Node.js, 43만 줄 코드, GitHub 23.6만 스타, 최소 2GB RAM (권장 8GB)
- ZeroClaw: Rust, 정적 바이너리, GitHub 2만 스타, **5~8MB RAM만 필요**, 출시 2주차

ZeroClaw의 핵심 장점은 극단적인 경량화로, $10짜리 하드웨어(BeagleBone 등)에서도 구동 가능하다. 정적 바이너리 방식이라 Node.js처럼 런타임 환경에 의존하지 않고 자체 실행된다. 메모리는 마크다운 파일 대신 데이터베이스를 사용한다.


**실사용 경험**: 실제로 OpenClaw를 통해 ZeroClaw를 Docker 컨테이너에 설치하고 Telegram 연동까지 성공했다. Claudson 4.6을 Venice API로 연결하여 작동시켰으며, 체감 응답 속도는 OpenClaw보다 빠르다.


**단점**: 스킬 설치가 매우 불편하다. ZIP 파일 업로드가 작동하지 않고, 마크다운 파일 공유도 실패하여 스킬 내용을 직접 붙여넣기해야 했다. 하트비트·크론잡·자기개선 같은 OpenClaw의 핵심 기능이 부재하며, **개발자/엔지니어 대상**이라는 인상이 강하다.


**결론**: "OpenClaw 승리". ZeroClaw는 기술적으로 인상적이지만, 비개발자 입장에서 스킬 관리 편의성과 자율 에이전트 기능 면에서 OpenClaw에 미치지 못한다. 다만 보안 강화 버전의 OpenClaw 대안으로서의 잠재력은 인정한다.
