## **종합 분석**


2026년 3월 15일 기준, OpenClaw("용랍스타/龍蝦")는 전 세계적으로 가장 뜨거운 AI 에이전트 플랫폼이다. 33개 영상이 하루 만에 올라왔고, 7개 이상의 언어권에서 콘텐츠가 생산되고 있다. 단순한 기술 유행이 아니라 AI 산업의 구조적 변화를 보여주는 현상이며, 자막 분석을 통해 다섯 가지 핵심 인사이트를 도출할 수 있다.


**첫째, OpenClaw 3.13은 "보는 AI"에서 "행동하는 AI"로의 전환점이다.** 3월 13일 공개된 이번 업데이트의 핵심은 Chrome DevTools 프로토콜을 통한 실제 브라우저 연결이다. 이전까지 AI 에이전트는 별도의 헤드리스 브라우저에서 빈 세션으로 작업했지만, 이제는 사용자가 로그인한 Gmail, GitHub, 내부 대시보드 등에 직접 접근할 수 있다. 이 변화는 단순한 기능 추가가 아니라, AI 에이전트가 "디지털 관찰자"에서 "디지털 행위자"로 격상되는 패러다임 전환이다.


**둘째, 중국에서 "국가 안보 차원"의 금지령이 내려졌다.** 중국 정부, 은행, 국유기업은 업무용 기기에서의 OpenClaw 설치를 공식 금지했으며, 군인 가족까지 포함되는 강력한 조치다. 한 선전 프로그래머는 설치 3일 만에 API 키 유출로 1.2만 위안(약 230만 원)의 직접 손실을 입었다. 흥미로운 점은 "설치 대행" 시장이 형성되어 29~599위안의 서비스가 성행했고, 이제는 "제거 대행" 시장으로 전환되고 있다는 것이다. 결국 돈을 번 것은 OpenClaw을 사용한 사람이 아니라 설치·제거 대행업자라는 아이러니가 발생했다.


**셋째, 오픈소스 생태계에서 "예의와 신뢰"의 가치가 재조명되고 있다.** 텐센트는 공식 기술 상점 ClawHub의 1만여 개 스킬 데이터를 무단 크롤링하여 SkillHub라는 복제 사이트를 만들었다. 코드 기여도, 기부금도 없이 속도 제한에 대한 불만까지 제기하여 OpenClaw 창시자 Peter Steinberger의 공개 비판을 받았다. 반면 바이두는 DuClaw를 통해 공식 협력 방식으로 클라우드 기반 OpenClaw 환경을 제공하며 칭찬을 받았다. 이는 오픈소스에서 기술적 역량보다 커뮤니티 신뢰가 더 중요하다는 교훈을 남긴다.


**넷째, AI 에이전트는 사람이 아닌 "사람이 쓰던 소프트웨어"를 먼저 대체하고 있다.** 한국 채널 '안될공학'의 분석이 인상적이다. AI에게 읽기 권한을 주면 비서, 쓰기 권한을 주면 대리인, 결제 권한을 주면 이미 직원이 된다. SaaS의 본질은 "사람의 손을 인터페이스에 연결하는 산업"인데, 에이전트는 UI를 건너뛰므로 UI 위에 세워진 SaaS는 위협받고, 데이터 위에 세워진 SaaS(Salesforce, Microsoft 등)는 오히려 AI를 내재화하여 강해진다. 이 분류가 향후 AI 시대 기업 가치 평가의 핵심 기준이 될 수 있다.


**다섯째, 메모리 문제는 OpenClaw의 아킬레스건이자 생태계 확장의 기회다.** 장시간 세션에서 에이전트가 이전 대화를 잊어버리는 문제가 가장 빈번한 불만이다. 창시자가 직접 추천한 Lossless Claw 플러그인이 해결책으로 부상했으며, 이는 OpenClaw 생태계가 코어 플랫폼과 써드파티 플러그인으로 분화·성장하고 있음을 보여준다.
